# 比升演示管理后台 - 产品设计规格

## 一、产品概述

### 1.1 产品定位
比升演示管理后台是比升技术官网的内部管理系统，主要用于管理演示预约、账号权限和交互式方案展示。

### 1.2 目标用户
- 比升技术内部员工
- 销售团队
- 产品演示人员
- 管理层

### 1.3 核心价值
- 集中管理预约演示数据
- 统一账号权限管理
- 提供交互式方案展示平台

## 三、详细功能设计

### 3.1 登录认证模块

#### 3.1.1 登录入口
- **位置**：官网首页右上角导航栏
- **样式**：文字链接"登录"，hover显示下划线
- **权限**：所有访客可见

#### 3.1.2 登录页面
- **路由**：`/login`
- **布局**：居中卡片式登录表单

**页面元素**：
| 元素 | 类型 | 说明 |
|------|------|------|
| Logo | 图片 | 比升技术Logo |
| 标题 | 文字 | "比升演示管理后台" |
| 账号输入框 | Input | placeholder: "请输入账号" |
| 密码输入框 | Input | placeholder: "请输入密码"，密码模式 |
| 记住我 | Checkbox | 可选，保持登录状态7天 |
| 登录按钮 | Button | 主按钮，点击验证登录 |
| 错误提示 | Text | 账号或密码错误时显示 |

**默认账号**：
- 账号：admin
- 密码：Bitsun1234

#### 3.1.3 认证流程
```
用户输入账号密码 → 前端验证非空 → 调用登录API → 
验证账号密码 → 成功：生成Token，跳转后台首页 → 
失败：显示错误提示
```

#### 3.1.4 会话管理
- Token存储：localStorage + Cookie
- Token有效期：7天（勾选记住我）/ 24小时（未勾选）
- 登出：清除Token，跳转登录页

---

### 3.2 账号管理模块

#### 3.2.1 用户列表页面
- **路由**：`/admin/accounts`
- **布局**：左侧菜单 + 右侧内容区

**列表字段**：
| 字段 | 说明 | 操作 |
|------|------|------|
| ID | 用户唯一标识 | - |
| 账号 | 登录账号 | - |
| 姓名 | 用户姓名 | - |
| 角色 | admin/user | - |
| 状态 | 启用/禁用 | 切换开关 |
| 创建时间 | 注册时间 | - |
| 最后登录 | 最近登录时间 | - |
| 操作 | 编辑/删除 | 按钮 |

**功能按钮**：
- 新增用户：打开新增用户弹窗
- 搜索：按账号/姓名搜索
- 筛选：按角色/状态筛选

#### 3.2.2 新增/编辑用户弹窗

**表单字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 账号 | Input | 是 | 唯一，字母数字组合 |
| 姓名 | Input | 是 | 中文或英文 |
| 密码 | Input | 是(新增)/否(编辑) | 至少8位，包含字母和数字 |
| 确认密码 | Input | 是(新增)/否(编辑) | 与密码一致 |
| 角色 | Select | 是 | admin/user |
| 状态 | Switch | 是 | 启用/禁用 |

#### 3.2.3 角色权限

| 角色 | 权限 |
|------|------|
| admin | 所有功能，包括账号管理 |
| user | 预约演示记录查看、交互式方案查看 |

### 3.3 预约演示记录模块

#### 3.3.1 记录列表页面
- **路由**：`/admin/demos`
- **数据来源**：`data/contact-submissions/` 目录下的JSON文件

**列表字段**：
| 字段 | 说明 |
|------|------|
| 序号 | 自动编号 |
| 姓名 | 提交人姓名 |
| 公司 | 公司名称 |
| 职位 | 职位（如有） |
| 手机 | 联系电话 |
| 邮箱 | 电子邮箱 |
| 感兴趣方案 | 勾选的方案 |
| 留言 | 备注留言 |
| 来源 | 提交页面 |
| 提交时间 | 表单提交时间 |
| 状态 | 未处理/已联系/已转化 |
| 操作 | 查看/标记状态 |

#### 3.3.2 详情弹窗
点击"查看"显示完整信息，包括：
- 所有表单字段详情
- 提交时间
- 处理记录（如有）

#### 3.3.3 状态管理
- 未处理（默认）：黄色标签
- 已联系：蓝色标签
- 已转化：绿色标签

#### 3.3.4 导出功能
- 支持导出为Excel/CSV格式
- 可选择导出范围（全部/筛选结果）

## 四、技术设计

### 4.1 技术栈
- **前端**：Next.js 16 + React 19 + TypeScript
- **样式**：CSS Modules
- **状态管理**：React Context + useState
- **数据存储**：JSON文件系统

### 4.2 目录结构
```
src/
├── app/
│   ├── login/
│   │   └── page.tsx              # 登录页面
│   └── admin/
│       ├── layout.tsx            # 后台布局
│       ├── page.tsx              # 后台首页
│       ├── accounts/
│       │   └── page.tsx          # 账号管理
│       ├── demos/
│       │   └── page.tsx          # 预约演示记录
│       └── solutions/

├── components/
│   ├── admin/
│   │   ├── Sidebar.tsx           # 侧边栏
│   │   ├── Header.tsx            # 顶部导航
│   │   └── ...

│       └── AuthProvider.tsx      # 认证上下文
```

### 4.3 API设计

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/login` | POST | 登录验证 |
| `/api/auth/logout` | POST | 登出 |
| `/api/auth/verify` | GET | 验证Token |
| `/api/admin/users` | GET | 获取用户列表 |
| `/api/admin/users` | POST | 新增用户 |
| `/api/admin/users/[id]` | PUT | 更新用户 |
| `/api/admin/users/[id]` | DELETE | 删除用户 |
| `/api/admin/demos` | GET | 获取预约演示列表 |
| `/api/admin/demos/[id]` | PUT | 更新状态 |

## 六、安全设计

### 6.1 认证安全
- 密码加密存储（bcrypt）
- Token有效期限制
- 登录失败次数限制（5次锁定30分钟）

### 6.2 权限控制
- 路由守卫：未登录跳转登录页
- API权限验证：验证Token和角色
- 前端权限控制：根据角色显示/隐藏功能

### 6.3 数据安全
- 敏感数据不暴露给前端
- API请求日志记录
- 定期数据备份
