## 二、场景一：全渠道销售

### 2.1 场景定位

**核心主张**：统一管理线上线下销售渠道，实现全渠道销售一体化

**目标客户**：

- 多渠道销售的消费品企业
- 线上线下融合的零售企业
- 分销与零售一体化的集团企业

### 2.2 核心痛点

| 痛点ID           | 痛点名称        | 描述                                 |
| -------------- | ----------- | ---------------------------------- |
| channel-silo   | 渠道割裂，协同困难   | 线上线下、直营分销渠道独立运营，各自一套系统，协同难。        |
| price-chaos    | 终端杂多，价格失控。  | 企业多业务模式多系统下，终端渠道价格体系无法影响，结果管理容易失控。 |
| policy-fail    | 政策落地，传导低效   | 渠道促销政策精细化不足，执行依靠人工传导，效果差           |
| market-respons | 市场变化快，响应速度慢 | 市场模式和渠道红利变化快，无法快速响应业务变化，把握机会。      |

### 2.3 解决方案架构

```
全渠道销售解决方案
├── 渠道中心
│   ├── 全渠道档案管理
│   ├── 上下游伙伴视图
│   └── 多维价格体系
├── 销售管理
│   ├── 多业务模式支撑
│   ├── 渠道协议与控销
│   └── 精细化业务控制
├── 订单履约
│   ├── 2B/2C自动化审单
│   ├── 智能拆单合单
│   └── 白盒化可运营寻源
└── 结算管理
    ├── 多场景分润配置
    ├── 多凭证自动生成
    └── 智能渠道对账
```

### 2.4 核心功能

#### 2.4.1 渠道中心

| 功能模块    | 功能亮点                     | 解决痛点            |
| ------- | ------------------------ | --------------- |
| 全渠道档案管理 | 统一管理线上店铺与线下门店档案          | 多渠道数据分散，管理成本高   |
| 上下游伙伴视图 | 整合销售客户与采购商档案，360度管理      | 客户信息分散，缺乏统一管理体系 |
| 多维价格体系  | 支持采购价、吊牌价、页面价、成交价等多维价格维护 | 价格体系混乱，利润管控失效   |

#### 2.4.2 销售管理

| 功能模块    | 功能亮点                | 解决痛点             |
| ------- | ------------------- | ---------------- |
| 多业务模式支撑 | 支持经销、代销、常规、直发等多种模式  | 集团多组织业务模式复杂，管理割裂 |
| 渠道协议与控销 | 基于渠道协议执行限品、限量等控制策略  | 渠道窜货、乱价难以管控      |
| 精细化业务控制 | 支持退货、额度、少收/溢收等字段级管控 | 2B业务场景复杂，风险高     |

#### 2.4.3 订单履约

| 功能模块       | 功能亮点                  | 解决痛点             |
| ---------- | --------------------- | ---------------- |
| 2B/2C自动化审单 | 智能识别并自动化审核各类订单        | 人工审单效率低，大促期间订单积压 |
| 智能拆单合单     | 基于仓库分布、物流成本、商品属性精细化策略 | 盲目拆单导致物流成本激增     |
| 白盒化可运营寻源   | 支持多指标配置与动态调整，透明化记录全过程 | 执行过程黑盒，难以追溯      |

#### 2.4.4 结算管理

| 功能模块    | 功能亮点                | 解决痛点            |
| ------- | ------------------- | --------------- |
| 多场景分润配置 | 灵活配置复杂的分润规则         | 分润规则复杂，人工计算易出错  |
| 多凭证自动生成 | 单笔销售订单自动还原生成多张结算凭证  | 业财数据不一致，财务工作量大  |
| 智能渠道对账  | 自动化完成渠道交易数据与财务数据的核对 | 渠道对账耗时耗力，差异难以查明 |

| ------ | ----- | -------------- |

| 订单处理效率 | 提升80% | 自动化审单，减少人工干预   |
| 对账效率   | 提升90% | 自动化对账，快速定位差异   |
| 客户满意度  | 提升30% | 统一客户视图，精准服务    |

## 三、场景二：一盘货运营

### 3.1 场景定位

**核心主张**：打破渠道壁垒，实现库存共享与智能调度

**目标客户**：

- 多渠道库存管理困难的企业
- 库存周转率低的企业
- 超卖与缺货并存的企业

### 3.2 核心痛点

| 痛点ID             | 痛点名称 | 描述                     |
| ---------------- | ---- | ---------------------- |
| inventory-silo   | 库存割裂 | 渠道库存割裂，线上线下、直营分销库存无法共享 |
| over-stock       | 超卖缺货 | 可销库存利用率低，超卖、缺货问题并存     |
| low-turnover     | 周转低效 | 多盘货下，库存周转率低，资金占用严重     |
| blind-allocation | 盲目分配 | 人工分配库存效率低，响应慢          |

### 3.3 解决方案架构

```
一盘货运营解决方案
├── 库存模型
│   ├── 四层库存模型
│   ├── 多状态库存分层
│   └── ATP可承诺库存
├── 货通场景
│   ├── 直营货通
│   ├── 分销货通
│   ├── 线下货通
│   ├── 线上一盘货
│   └── 全域一盘货
├── 共享和履约策略
│   ├── 库存精细共享
│   ├── 渠道动态分配
│   └── 全局最优履约
└── 预警管理
    ├── 效期预警
    ├── 库龄预警
    └── 异常预警
```

### 3.4 核心功能

#### 3.4.1 库存模型

| 功能模块     | 功能亮点                    | 解决痛点                    |
| -------- | ----------------------- | ----------------------- |
| 四层库存模型   | 物理库存、逻辑库存、可销库存、可用库存分层管理 | 库存状态单一，无法精准区分可用量        |
| 多状态库存分层  | 细分为可销、预占、在途、冻结等状态       | 库存状态单一，导致超卖或库存积压        |
| ATP可承诺库存 | 统筹计划、在制、在途库存，支持预售分配     | 库存visibility不足，无法准确承诺交期 |

#### 3.4.2 货通场景

| 功能模块  | 功能亮点           | 解决痛点            |
| ----- | -------------- | --------------- |
| 直营货通  | 直营门店间库存共享与调拨   | 门店库存割裂，畅销品缺货    |
| 分销货通  | 分销渠道间库存共享与调配   | 分销库存无法共享，资源浪费   |
| 线下货通  | 线下多业态门店库存共享    | 线下门店库存割裂，无法互相调拨 |
| 线上一盘货 | 线上多平台库存统一管理    | 多平台库存独立，超卖缺货并存  |
| 全域一盘货 | 线上线下全渠道库存共享与调度 | 全渠道库存割裂，资源利用率低  |

#### 3.4.3 共享和履约策略

| 功能模块   | 功能亮点                   | 解决痛点             |
| ------ | ---------------------- | ---------------- |
| 库存精细共享 | 支持按渠道、区域、品类等维度精细化共享配置  | 共享策略粗放，无法满足差异化需求 |
| 渠道动态分配 | 基于销售预测与库存状况，动态分配渠道库存   | 分配策略僵化，响应市场变化慢   |
| 全局最优履约 | 基于成本、时效、库存等多维度全局最优履约决策 | 履约策略局部优化，整体成本高   |

#### 3.4.4 预警管理

| 功能模块 | 功能亮点         | 解决痛点          |
| ---- | ------------ | ------------- |
| 效期预警 | 基于效期规则自动触发预警 | 效期商品损耗大，合规风险高 |
| 库龄预警 | 基于库龄规则自动触发预警 | 呆滞库存增加，资金占用严重 |
| 异常预警 | 库存异常实时预警     | 库存问题发现滞后      |

### 3.5 核心价值指标

| 指标     | 数值   | 说明             |
| ------ | ---- | -------------- |
| 可销库存放大 | 11倍  | 全渠道库存共享，放大可销库存 |
| 库存共享率  | 70%+ | 渠道间库存共享比例      |
| 库存周转降低 | 20天  | 库存周转天数降低       |
| 缺货率降低  | 25%  | 缺货率平均降低        |

### 3.6 适用行业

- 大美丽：鞋服时尚、日化美妆（多SKU、快周转）
- 大健康：医药流通、食品快消（效期管理）
- 大生活：家居家电、母婴用品（大件物流）

### 3.7 关联产品

- 星澜全渠道OMS（核心）
- 星智AI智能商品运营

## 四、场景三：商品运营智能

### 4.1 场景定位

**核心主张**：AI驱动的商品运营决策，优化库存与销售

**目标客户**：

- 商品运营效率低的企业
- 依赖人工经验决策的企业
- 需要精细化商品管理的企业

### 4.2 核心痛点

| 痛点ID             | 痛点名称       | 描述                              |
| ---------------- | ---------- | ------------------------------- |

| 库存自动同步 | 多平台库存实时同步，防止超卖 | 库存同步不及时，超卖客诉 |

| 审核自动处理 | 智能识别订单类型，自动化审核 | 人工审单效率低，大促积压 |
| 履约自动执行 | 自动匹配仓库、生成发货单   | 履约环节多，人工操作易错 |

#### 5.4.3 售后管理

| 功能模块    | 功能亮点                 | 解决痛点          |
| ------- | -------------------- | ------------- |
| 售后归因分析  | 拉通客户、仓库、实物质量数据，一站式分析 | 售后原因不明，无法改进   |
| 无头件智能匹配 | 基于策略自动匹配无头件包裹        | 无头件处理耗时耗力     |
| 退款自动处理  | 符合规则的退款自动处理          | 退款处理效率低，客户体验差 |

#### 5.4.4 风控管理

| 功能模块    | 功能亮点               | 解决痛点         |
| ------- | ------------------ | ------------ |
| 异常交易监控  | 大订单量下自动监控异常交易      | 人工监控无法覆盖全量订单 |
| 破价提醒熔断  | 实时监控价格异常，触发提醒或自动熔断 | 运营失误导致标价过低   |
| 库存共享防超卖 | 策略平衡超卖风险与库存利用率     | 共享库存易超卖导致客诉  |
